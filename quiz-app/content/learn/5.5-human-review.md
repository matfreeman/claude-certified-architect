## What is confidence calibration for human review?

As you automate more decisions with Claude — document classification, invoice extraction, content review — you need to know which decisions to trust and which to route to humans. The obvious approach is to use Claude's confidence scores: high confidence → accept, low confidence → review.

The problem is that aggregate accuracy numbers and self-reported confidence scores can both be misleading in ways that put incorrect outputs through to production. This lesson covers how to build human review workflows that catch those failures.

## Aggregate accuracy masks segment failures

A 97% overall accuracy rate sounds reliable. But look at what's underneath:

```
Overall accuracy:         97%

Broken down by segment:
  Standard invoices:      99.5% ✅
  Handwritten invoices:   78%   ⚠️
  Non-English invoices:   65%   ❌
  Multi-page invoices:    91%   ⚠️
```

If you reduce human review based on the 97% aggregate, you're silently automating a 65% error rate on non-English invoices.

**Always validate by document type and field segment before reducing human review coverage.** Aggregate numbers hide sub-population failures.

## Stratified sampling catches systematic errors

The natural instinct is to review low-confidence extractions and auto-accept high-confidence ones. The problem: systematic errors appear in *high-confidence* outputs. If Claude consistently misclassifies a specific payment term as "net-30" when it should be "net-60", it does so confidently every time. Those errors never enter your review queue.

```python
def create_review_sample(extractions: list) -> list:
    high_confidence   = [e for e in extractions if e["confidence"] == "high"]
    medium_confidence = [e for e in extractions if e["confidence"] == "medium"]
    low_confidence    = [e for e in extractions if e["confidence"] == "low"]
    
    # Sample from EACH group — high-confidence sample detects systematic errors
    return (
        random.sample(high_confidence, min(20, len(high_confidence))) +
        random.sample(medium_confidence, min(30, len(medium_confidence))) +
        list(low_confidence)   # Review all low-confidence
    )
```

Stratified sampling gives you coverage across all confidence levels, not just the uncertain ones.

## Field-level confidence routing

A single confidence score per document hides which fields are reliable. Field-level scores let you route selectively:

```python
extraction = {
    "invoice_number": {"value": "INV-2025-001", "confidence": "high"},
    "total_amount":   {"value": 15000.00,        "confidence": "high"},
    "vendor_address": {"value": "123 Main St...", "confidence": "medium"},
    "payment_terms":  {"value": "net-30",         "confidence": "low"},  # → human review
}
```

A human reviewer sees only the flagged fields, not the whole document. This reduces review time while still catching the uncertain extractions.

## Calibrating thresholds with labeled validation sets

Confidence thresholds need calibration against ground truth — the model's "high" confidence may not match your acceptable error rate for that field.

```
1. Collect labeled ground-truth extractions (100–500 documents)
2. Run the model on these documents
3. Compare model confidence levels to actual accuracy at each level
4. Set review thresholds based on actual error rates
5. Re-calibrate quarterly as document types change
```

A threshold set against labeled data is evidence-based. A threshold set by intuition ("high confidence seems reliable") is not.

## Key things to know

- Aggregate accuracy hides segment failures — always validate by document type, not just overall
- Stratified sampling is required because systematic errors appear in high-confidence outputs
- Field-level confidence routing is more efficient than document-level routing
- Thresholds must be calibrated against labeled validation sets, not intuition
- Re-calibrate regularly as document types and model behaviour drift over time

## Exam tips

- "97% accuracy, so we reduced review" → wrong without segment-level validation first

- "Our review queue is empty" → check high-confidence systematic errors, not just low-confidence volume
- The exam may ask how to detect systematic errors → stratified sampling from high-confidence extractions, reviewed against ground truth
