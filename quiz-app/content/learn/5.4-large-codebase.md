## What is context management in codebase exploration?

Exploring a large codebase is one of the most context-intensive tasks an agent can do. Every file read, every search result, every discovered connection adds to the context window. Without management, two things happen: the context fills up (triggering a hard limit), or it becomes so dense with intermediate discoveries that Claude starts losing track of earlier findings.

The strategies in this lesson are about keeping a long exploration session reliable from start to finish — not just the first few hundred turns.

## Signs of context degradation

Watch for these signals that context is getting overloaded:

- Agent gives inconsistent answers about findings from earlier in the session
- References "typical patterns" instead of the specific classes or functions found earlier
- Contradicts earlier conclusions ("this module has no dependencies" when it just listed them)
- Forgets which files were already analyzed

When you see these, context management is failing. The fix is one of the techniques below.

## Scratchpad files for persistent findings

Write key discoveries to a file as you find them — don't rely on the conversation history to hold them:

```
# Codebase Exploration Scratchpad

## Key Architecture
- Entry point: src/index.ts → imports from src/server.ts
- Auth flow: AuthMiddleware (src/middleware/auth.ts) → JWT validation → UserService
- Database: PostgreSQL via TypeORM, repository pattern in src/repositories/

## Files Related to Bug
- src/services/OrderService.ts (line 145 — duplicate charge logic)
- src/repositories/PaymentRepository.ts (missing transaction boundary)

## Unexplored (HIGH PRIORITY)
- src/webhooks/   — not yet read
- src/jobs/       — background jobs unclear
```

Write this with the `Write` tool, update it as findings accumulate, and reference it with `Read` instead of re-exploring. The file persists even if the conversation context gets compacted.

## Subagent delegation for focused exploration

Instead of having the main agent read hundreds of files and accumulate all that output in its own context, delegate specific questions to subagents:

```python
exploration_tasks = [
    "Find all test files and report what they cover",
    "Trace the refund flow from API endpoint to database",
    "Identify all services that depend on UserService",
]

# Each subagent reads files, runs searches, returns a concise summary
# Main agent accumulates summaries — not raw file contents
```

The main agent stays uncluttered. Each subagent takes on the verbose discovery for its specific question and returns a dense summary.

## Summarise before spawning subagents

After an initial exploration phase, compress what was learned before starting the detailed work:

```python
# After first exploration phase
summary = "Architecture: Express API, PostgreSQL + TypeORM, JWT auth. Key issue area: OrderService."

# Inject into subagent prompts
subagent_prompt = f"""
Context from prior exploration:
{summary}

Your task: Trace the payment processing flow in detail.
Start from src/api/payments.ts and follow the dependency chain.
"""
```

This gives each subagent the context it needs without passing the full exploration transcript.

## The daisy-chaining anti-pattern

Passing full conversation logs or raw agent transcripts from one agent to the next scales token costs badly and reduces reliability. Each agent in the chain receives more context than the one before, and quality degrades.

The fix is structural: modify upstream agents to return *structured findings*, not verbose reasoning. Use coordinator-managed scratchpads or shared files so downstream agents query only what they need.

## Crash recovery using state manifests

For long multi-agent workflows, maintain a manifest that captures enough state to resume:

```python
agent_state = {
    "agent_id": "web-searcher",
    "completed_queries": ["AI music 2025", "AI visual arts 2024"],
    "pending_queries": ["AI film production 2025"],
    "results_file": "~/.claude/research/web_results.json",
    "status": "in_progress"
}

# Write to a known location
write_json("~/.claude/research/manifest.json", agent_state)

# On resume: coordinator loads manifest and injects into prompt
resume_prompt = f"Resume from manifest: {json.dumps(manifest)}"
```

## Key things to know

- Context degradation shows as inconsistency, not as an explicit error — watch for contradictions and vague references
- Scratchpad files persist beyond `/compact` — they're the right place for findings that must survive
- Subagent delegation keeps the main agent's context lean; subagents handle the verbosity
- Daisy-chaining raw transcripts is an anti-pattern — upstream agents should return structured summaries
- State manifests enable crash recovery; without them, a failed agent loses all progress

## Exam tips

- "Agent starts referencing 'typical patterns' instead of specific findings" → context degradation — use scratchpad files
- "Main agent context fills after reading 200 files" → delegate file reading to subagents
- `/compact` compresses conversation history; scratchpad files on disk survive compaction
- Daisy-chaining full transcripts → the exam treats this as an anti-pattern; the fix is structured summaries
